# Security Policy

## Supported Versions

Cardiff is in an early pre-release state.
Only the latest `0.1.x` work and the most recent project state are considered supported for security fixes.

| Version | Supported |
| --- | --- |
| 0.1.x | Yes |
| Older drafts | No |

## Reporting a Vulnerability

Do not open public issues for suspected security problems.

Report vulnerabilities privately to `zecalifornia@up.edu.ph` and include:

- the affected file, component, or workflow
- reproduction steps or a proof of concept
- the expected impact
- any suggested mitigation, if available

## What to Expect

Best-effort response targets:

- acknowledgment within 72 hours
- initial triage within 7 calendar days
- remediation planning after triage once impact is confirmed

Complex issues may take longer, but status updates should be provided when practical.

## Responsible Disclosure

- Please avoid public disclosure until a fix or mitigation is available.
- Please avoid testing that would damage data, disrupt service, or violate privacy.
- Reporter credit can be given after resolution unless anonymity is requested.
